Layout

Main header and footer

Header and footer are separate applications that contain some of the styles in this toolkit and are not documented here. This documentation covers anything else within them:


+-header----------------------------------------------+
|/////////////////////////////////////////////////////|
+-----------------------------------------------------+
+-main_page_area--------------------------------------+
|                                                     |
|  this-documentation                                 |
|                                                     |
+-----------------------------------------------------+
+-footer----------------------------------------------+
|/////////////////////////////////////////////////////|
+-----------------------------------------------------+

The main page area is split in two sibling sections (elements): page header and page content:


+-page_header-----------------------------------------+
|                                                     |
+-----------------------------------------------------+
+-page_content----------------------------------------+
|                                                     |
+-----------------------------------------------------+

The content wrapper

The general page layout has some special margins and paddings for branding purposes. A full-width-wrapper mixin is provided to take care of this:


.some-content-wrapper {
  @include full-width-wrapper;
}

The full-width-wrapper is needed only twice: one for the page header, one for the page content. In the page header, the full-width-wrapper is nested inside. In the page content, the full-width-wrapper wraps the content. This is better explained below.

Page header (and breadcrumbs)

Contains a full-width-wrapper element (see above), that allows for the header to have a full width background color while maintaining alignment with the rest of the page elements such as header and footer. Page header contents are the nested within: breadcrumbs (optional) and header as an h1 element.

Home > Research > Articles & databases

Articles & databases


.my-page-header {
  @include page-header;
}

.my-breadcrumb {
  @include breadcrumbs;
}

.my-content-wrapper {
  @include full-width-wrapper;
}
    

<div class="my-page-header">
  <div class="my-content-wrapper">
    <div class="my-breadcrumb"><a href="#">Home</a> > <a href="#">Research</a> > Articles &amp; databases</div>
    <h1>Articles &amp; databases</h1>
  </div>
</div>

Page content

Is wrapped by a full-width-wrapper element that takes care of the general page padding and centering (see above). Inside this wrapper is a row/column layout that contains the entirety of any given page's main contents.


+-wrapper---------------------------------------------+
| +-row_and_cols------------------------------------+ |
| |                                                 | |
| |                                                 | |
| +-------------------------------------------------+ |
| +-row_and_cols------------------------------------+ |
| |                                                 | |
| |                                                 | |
| +-------------------------------------------------+ |
| +-row_and_cols------------------------------------+ |
| |                                                 | |
| |                                                 | |
| +-------------------------------------------------+ |
+-----------------------------------------------------+

The rows and columns themselves are based on a simple, mobile-ready row-4-column pattern that can be combined as needed. Existing column size mixins are column-full, column-half, column-one-quarter, and column-three-quarters.

NOTE: avoid nesting rows inside columns as the system does not support it for the time being. It is preferred to use multiple rows if you require a more elaborate layout.

Forms fields should be wrapped by a fieldset tag.

Text fields


.my-text-field {
  @include text-fieldset;
}
    

<fieldset class="my-text-field">
  <label for="keyword-text">Title or description</label>
  <input id="keyword-text" type="text">
</fieldset>

    <option value="1">One</option>
    <option value="2">Two</option>
    <option value="3">Three</option>
    <option value="…">…</option>
  </select>
</fieldset>

Alphabetical filters

Not a form element per se but still used within search forms. Selected element is replaced with span. The "Any" selection has a long class.

Any # A B C … 

.alphabetical {
  @include alphabetical-filter;
}

NOTE: Maybe replace a tag with button?


<fieldset class="alphabetical">
  <label>Name starts with</label>
  <span class="long">Any</span>
  <a href="#" title="Show only resources with name starting with a number">#</a>
  <a href="#" title="Show only resources with name starting with A">A</a>
  <a href="#" title="Show only resources with name starting with B">B</a>
  <a href="#" title="Show only resources with name starting with C">C</a>
  <a href="#" title="Show only resources with name starting with …">…</a>
</fieldset>

Radiobutton/checkbox groups

For multiple browser-compatibility reasons we avoid the legend tag and use a span tag with a nested strong tag.

Accessible from 

.radiogroup {
  @include radiobutton-fieldset;
}

NOTE: In this case the input tags are wrapped by the label tag.


<fieldset class="radiogroup">
  <span><strong>Accessible from</strong></span>
  <label for="available-every">
    <input id="available-every" type="radio" name="available" value="any">Anywhere
  </label>
  <label for="available-home">
    <input id="available-home" type="radio" name="available" value="home">Home with a library card
  </label>
  <label for="available-library">
    <input id="available-library" type="radio" name="available" value="library">A library branch
  </label>
</fieldset>

General tables

Search results summary

Presents a comma-separated list of search terms or filters along with the number of results. The result count has a special span tag (should be number-formatted e.g.: 1,232). Each applied facet or term has an anchor element to remove the facet and refresh the search results.

NOTE: Avoid adding a space between the facet and the remove link so that it won't wrap to the next line.

1,807 resources found for Young Adult× , Fiction× . 

.results-summary {
  @include results-summary;
}

NOTE: Maybe replace a tag with button?


<div class="results-summary">
  <span class="results-count">1,807</span> resources found for
  <span class="facet">
    <strong>Young Adult<a href="#" title="Remove Young Adult filter">×</a></strong>
  </span>,
  <span class="facet">
    <strong>Fiction<a href="#" title="Remove Fiction filter">×</a></strong>
  </span>.
</div>
