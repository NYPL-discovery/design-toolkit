Forms fields should be wrapped by a fieldset tag.

Text fields

Text fields can include (but not rely on) a placeholder that provides additional information on the type of input it expects.


.my-text-field {
  @include text-fieldset;
}
    

<fieldset class="my-text-field">
  <label for="keyword-text">Title or description</label>
  <input id="keyword-text" type="text" placeholder="Enter keywords">
</fieldset>

Select boxes


.my-select-box {
  @include select-fieldset;
}
    

<fieldset class="my-select-box">
  <label for="topics">Topics</label>
  <select id="topics" name="topics">
    <option value="1">One</option>
    <option value="2">Two</option>
    <option value="3">Three</option>
    <option value="…">…</option>
  </select>
</fieldset>

Alphabetical filters

Not a form element per se but still used within search forms. This is a list of buttons that apply an alphabetical/number-type filter. Currently applied filter is disabled. The "Any" (or multi-character) selection has a long class. 


.alphabetical {
  @include alphabetical-filter;
}
    

<fieldset class="alphabetical">
  <label>Name starts with</label>
  <button name="az-filter" disabled="true" class="long">Any</button>
  <button name="az-filter" title="Show only resources with name starting with a number">#</button>
  <button name="az-filter" title="Show only resources with name starting with A">A</button>
  <button name="az-filter" title="Show only resources with name starting with B">B</button>
  <button name="az-filter" title="Show only resources with name starting with C">C</button>
  <button name="az-filter" title="Show only resources with name starting with …">…</button>
</fieldset>

Radiobutton/checkbox groups

For multiple browser-compatibility reasons we avoid the legend tag and use a span tag with a nested strong tag.

Accessible from 

.radiogroup {
  @include radiobutton-fieldset;
}

NOTE: In this case the input tags are wrapped by the label tag.


<fieldset class="radiogroup">
  <span><strong>Accessible from</strong></span>
  <label for="available-every">
    <input id="available-every" type="radio" name="available" value="any">Anywhere
  </label>
  <label for="available-home">
    <input id="available-home" type="radio" name="available" value="home">Home with a library card
  </label>
  <label for="available-library">
    <input id="available-library" type="radio" name="available" value="library">A library branch
  </label>
</fieldset>

Searchable field sets

Searchable field sets allow the user to see many non-exclusive options at the same time while also searching within the option list. Use searchable facets when the list of potential selectable items is too large (i.e. more than a dozen or so). The text field should autocomplete (with result count) as the user types in it. By default, show the top 10 items sorted by the amount of results that will be returned.

The general structure is: a .facet-search that reflects the general structure of a text field element followed by a facet-list of items, each with the following structure:


<label for="my-facet-value-id">
  <span class="facet-label">
    <input id="my-facet-value-id" type="checkbox" name="my-name" value="my-value">
    The facet text
  </span>
  <span class="facet-count">[estimated result count]</span>
</label>

A “Show X more” link at the end of the list allows for the user to display X more items to select from.

NOTE: Place previously checked items first.

Show 10 more subjects 

.my-searchable-facet {
  @include searchable-fieldset;
}
    

<fieldset class="my-searchable-facet">
  <div class="facet-search">

    <input id="subject-text" type="text" placeholder="Enter keywords">
  </div>
  <div class="facet-list">
