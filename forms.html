Forms

Go to top

Forms fields should be wrapped by a fieldset tag.

Omni search

Omni search is a special fieldset contained in its own row, in an offsetted column. (See Layout for details.) The fieldset itself contains three child elements: a submit button, a span tag of class .nypl-omni-fields and a text field. The .nypl-omni-fields element contains a hidden label (for screen readers only) and a select box with the applicable options.

NOTE: The text field is aria-labeledby the button.


.nypl-omnisearch
    

<fieldset class="nypl-omnisearch">
  <input type="submit" id="nypl-omni-button" value="Search">
  <span class="nypl-omni-fields">
    <label for="search-by-field">Search in</label>
    <select id="search-by-field"><option value="all">All fields</option><option value="title">Title</option><option value="contributor">Author/Contributor</option><option value="subject">Subject</option><option value="series">Series</option><option value="call_number">Call number</option></select>
  </span>
  <input type="text" aria-labelledby="nypl-omni-button" value="">
</fieldset>

Text fields

Text fields can include (but not rely on) a placeholder that provides additional information on the type of input it expects.


.nypl-text-fieldset
    

<fieldset class="nypl-text-fieldset">
  <label for="keyword-text">Title or description</label>
  <input id="keyword-text" type="text" placeholder="Enter keywords">
</fieldset>

Alphabetical filters

Not a form element per se but still used within search forms. This is a list of buttons that apply an alphabetical/number-type filter. Currently applied filter is disabled. The "Any" (or multi-character) selection has a nypl-long class. 


.nypl-alphabetical-filter
    

<fieldset class="nypl-alphabetical-filter">
  <label>Name starts with</label>
  <button name="az-filter" disabled="true" class="nypl-long">Any</button>
  <button name="az-filter" title="Show only resources with name starting with a number">#</button>
  <button name="az-filter" title="Show only resources with name starting with A">A</button>
  <button name="az-filter" title="Show only resources with name starting with B">B</button>
  <button name="az-filter" title="Show only resources with name starting with C">C</button>
  <button name="az-filter" title="Show only resources with name starting with …">…</button>
</fieldset>

Radiobutton/checkbox groups

For multiple browser-compatibility reasons we avoid the legend tag and use a span tag of class nypl-label.

Accessible from 

.nypl-radiobutton-fieldset

NOTE: In this case the input tags are wrapped by the label tag.


<fieldset class="nypl-radiobutton-fieldset">
  <span class="nypl-label">Accessible from</span>
  <label for="available-every">
    <input id="available-every" type="radio" name="available" value="any">Anywhere
  </label>
  <label for="available-home">
    <input id="available-home" type="radio" name="available" value="home">Home with a library card
  </label>
  <label for="available-library">
    <input id="available-library" type="radio" name="available" value="library">A library branch
  </label>
</fieldset>

Searchable field sets

Searchable field sets allow the user to see many non-exclusive options at the same time while also searching within the option list. Use searchable facets when the list of potential selectable items is too large (i.e. more than a dozen or so). The text field should autocomplete (with result count) as the user types in it. By default, show the top 10 items sorted by the amount of results that will be returned.

The general structure is: a nypl-facet-search that reflects the general structure of a text field element followed by a nypl-facet-list of items, each with the following structure:


<label for="my-facet-value-id">
  <span>
    <input id="my-facet-value-id" type="checkbox" name="my-name" value="my-value">
    The facet text
  </span>
  <span class="nypl-facet-count">[estimated result count]</span>
</label>

The facet count is optional but must be of class nypl-facet-count.

A “Show X more” link at the end of the list allows for the user to display X more items to select from.

NOTE: Place previously checked items first.

Show 10 more subjects 

.nypl-searchable-fieldset
.nypl-facet-search
.nypl-facet-list
